# Контекст проекта: транскрипция лекций по численным методам

> Этот файл — для возобновления работы (в т.ч. новой сессией Claude Code).
> Описывает окружение, скрипты, принятые решения и как обработать новые лекции.

## Цель
Сделать текстовые транскрипции видеолекций (18–33) по курсу «Численные методы»
и на их основе собирать ключевую информацию / конспекты.

- Рабочая папка: `/home/tema/Документы/chm`
- Видео: `lect/Лекция NN. *.mp4` (16 файлов, лекции 18–33)
- Готовые транскрипты: `transcripts/<имя видео>.txt` (таймкоды `[Ч:ММ:СС]`)

## Окружение (уже настроено)
- GPU: **NVIDIA GTX 1070 Ti** (Pascal, 8 ГБ). 24 ядра CPU, 30 ГБ RAM.
- Python venv: `.venv` (python 3.14). Установлено:
  `imageio-ffmpeg`, `faster-whisper`, `nvidia-cublas-cu12`, `nvidia-cudnn-cu12`,
  `socksio`, `huggingface_hub`.
- `ffmpeg` — статический бинарник из `imageio_ffmpeg.get_ffmpeg_exe()` (без sudo).

### Два важных подводных камня
1. **CUDA-библиотеки**: перед запуском нужно добавить в `LD_LIBRARY_PATH` пути
   `.venv/lib/python3.14/site-packages/nvidia/cublas/lib` и `.../nvidia/cudnn/lib`.
2. **Прокси**: в окружении задан `ALL_PROXY=socks://...`, который ломает httpx при
   скачивании модели с HuggingFace (схема `socks://` не поддерживается).
   Решение — запускать python с `env -u ALL_PROXY -u all_proxy -u HTTPS_PROXY
   -u https_proxy -u HTTP_PROXY -u http_proxy` (прямое соединение работает).

## Параметры распознавания (решения)
- Модель **`medium`**, `device="cuda"`, `compute_type="float32"`.
  (float16 на Pascal не поддерживается; large-v3 точнее по терминам, но в ~2,5–3×
  медленнее — для понимания/конспектов medium достаточно.)
- `language="ru"`, `beam_size=5`, `vad_filter=True`, `initial_prompt` со списком
  терминов курса (уменьшает ошибки в терминологии).
- Скорость: ~10–14 мин на лекцию (≈15× реального времени). GPU ~62–64 °C под 100%.

## Скрипты
- `batch_transcribe.py` — грузит модель один раз и прогоняет все `lect/*.mp4`,
  кладёт `.txt` в `transcripts/`. Пишет через `.tmp` + атомарная замена.
  **Пропускает уже готовые** файлы → можно перезапускать для продолжения.
  Лог: `/tmp/batch.log`.
- `fix_terms.py` — **идемпотентный** пост-процессор: чинит систематические ошибки
  ASR в терминах (сплавен→сплайн, интерпульционный→интерполяционный, проболический→
  параболический, поляном→полином, бактериал→факториал и т.д.). Безопасно
  запускать повторно. Запуск: `.venv/bin/python fix_terms.py` (по всей папке).
- `transcribe_full.py`, `transcribe_test.py`, `gputest.py` — разовые/отладочные.

## Как обработать НОВЫЕ лекции
1. Положить `.mp4` в `lect/`.
2. Запустить (из `/home/tema/Документы/chm`):
   ```bash
   SP=.venv/lib/python3.14/site-packages
   export LD_LIBRARY_PATH="$PWD/$SP/nvidia/cublas/lib:$PWD/$SP/nvidia/cudnn/lib:$LD_LIBRARY_PATH"
   env -u ALL_PROXY -u all_proxy -u HTTPS_PROXY -u https_proxy -u HTTP_PROXY -u http_proxy \
     .venv/bin/python batch_transcribe.py
   .venv/bin/python fix_terms.py
   ```

## Известные особенности / нюансы
- **Баг сортировки** в `batch_transcribe.py`: для имён с подчёркиваниями и хвостовой
  цифрой (Л24 «…к_2», Л26 «…_2») ключ сортировки раздувается (242, 262), поэтому
  порядок обработки не строго по номерам. На полноту НЕ влияет — берутся все `*.mp4`.
- Качество medium: смысл и математика передаются полностью; термины — «черновик»
  (часть чистится `fix_terms.py`). Формулы распознаются фонетически, но позиционно
  верно (напр. `аж ит` = h_i, «м большое и плюс первое» = M_{i+1}).
- Лекция 18 называется «Локальное сглаживание», но по содержанию — про сплайны
  (тема локального сглаживания в записи фактически не раскрыта).
