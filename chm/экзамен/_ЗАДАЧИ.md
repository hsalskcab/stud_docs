# Какие задачи будут на экзамене

`🧭` [Шпаргалка](_ШПАРГАЛКА.md) · [Определения](_ЭКЗАМЕН_тройка.md) · [Мелкие вопросы](_МЕЛКИЕ_ВОПРОСЫ.md)

> Источник: формулировки «перекличек» из лекций — преподаватель прямо говорит
> «вот так у вас будет формулироваться в перекличках» (Л31 [0:24:08]), а экзамен
> строится на том же материале. Задачи нужны **на повышение оценки** (Л33 [2:47:40]:
> «кто повышать будет — задачи решать»). Для **тройки** задачи не обязательны —
> хватает определений и формулировок (см. `_ЭКЗАМЕН_тройка.md`).

---

## Тип A. Выписать разностную схему для уравнения 2-го порядка
**Формулировка (Л27 [0:08:09]):** «выписать разностную схему для уравнения
второго порядка».
Что делать: заменить производные разностными отношениями
(y'' → (yᵢ₋₁ − 2yᵢ + yᵢ₊₁)/h², y' → разностью), расписать в узлах, добавить
краевые условия. ⚠️ Частая ошибка: расписывать коэффициенты не в той точке —
в коэффициентах должно стоять значение в xᵢ. [Л26 0:17:48–0:19:00]

## Тип B. Выписать систему разностных уравнений + прогонка
**Формулировка (Л26 [0:14:55–0:16:36]), две подзадачи:**
1. **Просто выписать** систему разностных уравнений — ничего не преобразуя.
2. **Преобразовать** её к **трёхдиагональному** виду (первое и последнее
   уравнения, краевые условия) и **выписать алгоритм прогонки** — показать,
   где стоят коэффициенты прогонки.
⚠️ Ловушка из переклички: не преобразовывать систему в задаче 1 (это требуется
только в задаче 2). И обязательно пояснять, что такое p(x), α₀, α₁ — не оставлять
«голые» три уравнения без пояснений (иначе «плюс со знаком вопроса»).

## Тип C. Исследовать схему на устойчивость методом гармоник
**Формулировка (Л31 [0:24:08]):** «Исследовать на устойчивость [по начальным
данным] разностное уравнение, соответствующее дифференциальному уравнению
uₜ + c·uₓ = 0 (c>0), на [заданном] шаблоне».
Что делать: записать разностное уравнение на указанном шаблоне (вперёд/назад/
центральная разность), подставить гармонику yⱼⁿ = λⁿ·e^{ikjh}, найти λ
(множитель перехода), проверить **|λ| ≤ 1** — если да, устойчива (возможно,
условно — тогда выписать условие на шаги, типа условия Куранта). [Л31 0:22:31–0:25:01]

## Тип D. Исследовать устойчивость алгебраическим методом / принципом максимума
**(Л30):** для краевой задачи (например y'' = …) показать устойчивость через
**диагональное преобладание** матрицы или через **принцип максимума** (оценка
максимума решения через данные, без матриц). Знаки вопроса в перекличке ставились
там, где «нет ни одного слова» — нужен словесный анализ коэффициентов схемы.
[Л30 0:35:08, 1:16:59, 1:32:14]

## Тип E. Найти решение в заданной точке (метод сеток, сквозной пересчёт)
**Формулировка (Л28 [2:15:37]):** «найти решение u в точке (например 1; 2,5)».
Что делать: построить разностную схему в индексах для уравнения переноса,
задать начальное (u на нулевом слое) и граничное условия, организовать
**сквозной пересчёт** — слой за слоем вычислить значения до нужной точки.
[Л28 2:46:10–2:53:22]

## Тип F. Метод Рунге–Кутты: выписать и найти константы
**(Л22–23):** по определению метода выписать k₁, k₂, …, приближённое решение
как линейную комбинацию; составить уравнения для констант из условия минимума
погрешности (совпадение рядов Тейлора). Узнаётся почти на любом экзамене
(Л23 [0:24:44]: «метод Рунге–Кутты идёт всегда»).

## Тип G. Аппроксимация оператора на «экзотическом» шаблоне ⭐
**(Л33 [2:48:18–2:48:35]):** шаблоны могут быть «достаточно экзотические,
сложные», но на них **надо уметь писать разностные операторы** и оценивать
порядок аппроксимации (разложение в ряд Тейлора, главный член погрешности).
Преподаватель называет это «квинтэссенцией курса».

## Сводка: на что делать упор
| Тема | Тип задачи |
|------|-----------|
| Краевая задача, прогонка | B (выписать систему + прогонка) |
| Уравнение 2-го порядка | A (разностная схема) |
| Уравнение переноса | C (гармоники), E (сквозной пересчёт) |
| Устойчивость | C, D |
| Задача Коши | F (Рунге–Кутты), Эйлер |
| Любой шаблон | G (аппроксимация оператора) |

**Главное для повышения (слова преподавателя, Л33 [2:47:57–2:48:35]):**
понимать на уровне идеи **аппроксимацию**, **устойчивость**, схему **с весами**
и **аппроксимацию разностных операторов на шаблонах**.
